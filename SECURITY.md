# Security policy

## Supported versions

Security fixes are provided for the latest `0.4.x` release and current `main`. Older releases are unsupported after a newer patch is published.

## Report a vulnerability

Do not open a public issue. Use GitHub private vulnerability reporting for this repository, or contact the maintainer privately through GitHub.

Include the affected version/commit, impact, reproduction steps, and any proposed mitigation. See [`docs/SECURITY.md`](docs/SECURITY.md) for the technical trust model.
